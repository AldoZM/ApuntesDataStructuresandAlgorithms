\chapter[\emj{рҹҢІ}~Fenwick Tree / BIT (Binary Indexed Tree)]{\emj{рҹҢІ}~Fenwick Tree / BIT (Binary Indexed Tree)}

\begin{dsaquees}

Una versiГіn sГәper compacta del Segment Tree para sumas de prefijo рҹ’°.
Imagina que quieres el saldo acumulado hasta el dГӯa 7, pero los montos
cambian a diario.

El Fenwick Tree guarda sumas parciales inteligentes: cada posiciГіn del
array "cubre" un bloque de tamaГұo potencia de 2, definido por su bit
menos significativo. Para sumar hasta i, saltas hacia atrГЎs quitando ese
bit; para actualizar, saltas hacia adelante sumГЎndolo. Ambos recorridos
son $O(\log n)$ y el cГіdigo cabe en cuatro lГӯneas.

\end{dsaquees}

\begin{dsaotraforma}

Un Fenwick Tree (o Binary Indexed Tree, BIT) es una estructura que
calcula sumas de prefijo y actualizaciones de punto en $O(\log n)$,
usando solo un array de tamaГұo $n+1$ (mucho menos que el $4n$ del Segment
Tree).

Su magia es el operador \verb|i & (-i)|, que aГӯsla el bit menos
significativo de $i$ y define el tamaГұo del bloque que cada Гӯndice
resume:
\begin{itemize}
\item \textbf{Query prefijo(i)}: parte de $i$ y resta el bit menos
      significativo hasta llegar a 0, acumulando.
\item \textbf{Update(i, delta)}: parte de $i$ y SUMA el bit menos
      significativo hasta pasar $n$, propagando el cambio.
\end{itemize}

Una suma de rango $[l, r]$ se obtiene como $\mathrm{prefijo}(r) -
\mathrm{prefijo}(l-1)$.

\end{dsaotraforma}

\begin{dsadiagrama}
ГҚndices (1-based) y quГ© bloque cubre cada uno (por lowbit):

  i:     1    2    3    4    5    6    7    8
  cubre: [1] [1-2][3] [1-4][5] [5-6][7] [1-8]

  lowbit(6)=2 -> el 6 cubre 2 elementos: [5,6]
  lowbit(8)=8 -> el 8 cubre 8 elementos: [1..8]

                  suma tree[7]+tree[6]+tree[4]

\end{dsadiagrama}

\begin{lstlisting}
CГ“MO FUNCIONA (1-indexado)
- lowbit(i) = i & (-i) aГӯsla el bit menos significativo.

в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

#include <vector>
using namespace std;

struct Fenwick {
    int n; vector<long long> tree;
    Fenwick(int n) : n(n), tree(n + 1, 0) {}   // 1-indexado

    void update(int i, long long delta) {
        for (; i <= n; i += i & (-i)) tree[i] += delta;
    }
    long long query(int i) {                   // suma de 1..i
        long long s = 0;
        for (; i > 0; i -= i & (-i)) s += tree[i];
        return s;
    }
    long long rango(int l, int r) {            // suma de l..r
        return query(r) - query(l - 1);
    }
};

\begin{dsaentrevistas}

вң… Fenwick vs Segment Tree: si solo necesitas sumas de prefijo con
   updates de punto, Fenwick gana (cГіdigo corto, menos memoria, mГЎs
   rГЎpido en la prГЎctica). Para mГӯnimo/mГЎximo o updates de rango, ve al

рҹ’Ў "Count of Smaller Numbers After Self" (LeetCode 315): se resuelve con

   inversiones.

вҡ пёҸ Es 1-indexado: el Гӯndice 0 no se usa. Si tu array es 0-based, suma 1
   al indexar o te quedas en un bucle infinito con \verb|i & (-i)|.

рҹ”‘ Conteo de inversiones: recorre de derecha a izquierda y consulta
   cuГЎntos elementos menores ya insertaste. Es EL uso estrella del BIT en
   entrevistas.

\end{dsaentrevistas}

\begin{dsaresumen}

вҖў BIT: sumas de prefijo + update de punto en O(log n).
вҖў Usa \verb|i & (-i)| (bit menos significativo) para saltar bloques.
вҖў query: resta el bit; update: suma el bit.
вҖў Rango [l,r] = prefijo(r) вҲ’ prefijo(lвҲ’1).
вҖў Es 1-indexado; array de tamaГұo n+1 (mГЎs ligero que Segment Tree).
вҖў Estrella para contar inversiones y "smaller after self".
